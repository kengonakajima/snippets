VCEとは何か?

VCEとは、"Virtual Community Engine" の略で、 サーバー集中型のネットワークゲーム(オンラインゲーム)や、 コミュニケーションツールを作るための基礎となる、 通信ミドルウェアです。 コミュニティエンジン株式会社 により開発されています。
VCEは以下のような部分から成りたっています。

  • Linuxサーバー用ライブラリ libvce.a
  • Windowsクライアント用ライブラリ vcewin32.lib
  • Ruby言語用ラッパーライブラリ vce.so
  • ドキュメント
  • CとRubyによるサンプルコード

VCEが提案するのは、 高負荷に耐えるコンテンツのサーバを「安全、簡単、迅速に」開発できる環境です。 低コストで、高いTCP/IP負荷に耐えるクラスタリングサーバを実装するための ライブラリやツールが含まれています。 コミュニティエンジンでは、サーバ集中型コンテンツの中でも、 特に性能とコストに対してシビアな要求のあるネットワークゲーム市場を 主ターゲット市場として選び、 ゲームクリエイターの自由な発想をすぐに形に変えることができる 一連の開発環境群を提供していきます。



VCE の技術的特徴

 VCE の技術的特徴は、以下に集約されます。
  • 必要最小限の機能を実装し、高速に動作する
    VCEは必要最小限の機能を実装し、 堅牢・高速に稼働することを目指してデザインされています。
  • シングルスレッド・サーバ
    VCEでは、コールバックを利用した自前のスレッディングで最高の 速度を追求します。 CPUが1個〜2個の安価なマシンではカーネルスレッドを多数使うよりも、 このようなアプローチのほうが高速です。
  • 「中継サーバー」を使ったクラスタリング
    多数のクライアントからの接続を、中継サーバを介して束ねることで、 高負荷状態における、UNIXのシステムコール呼びだし回数を劇的に減らします。 さらに、クラスタ・ノード間のセッション移動も、 アプリケーションプログラムから意識することなくできます。 この中継サーバは、暗号(SSL)、復号の機能も同時に持っているので、 本体サーバをアプリケーションのロジック処理のためにフル活用できます。
  • Diffie-Hellman/Blowfish を使った暗号化
    大量のデータを送受信する場合、暗号化の処理速度がネックになりますが、 その処理を中継サーバで行なうことによってシングルスレッドサーバの 性能を最大化します。 VCEでは、Diffie-Hellman + Blowfish/Rijndael の組みあわせで 強力で高速な暗号通信を可能にしています。
  • 高級言語API
    アイデアをすぐ形にする、という点ではC/C++言語は スクリプト言語にはかないません。 VCEでは、Ruby言語やKylixの API を提供していますので、 直感的なサーバ開発が可能です(その場合の性能低下は20%〜80%程度)。 サーバのプロトタイピングや、ライトなコンテンツに向きます。
  • 豊富な上位ライブラリ
    VCE を利用したアプリケーション開発をさらに効率的にするために、 「ソースコード共有」方式で各種上位ライブラリを提供します。 ソースコード共有により、柔軟で迅速なカスタマイズが可能となります。 上位ライブラリには現在、 "mm-suite","pp-suite","lobby-suite" があります。

ダウンロード

現在、本サイトでは、 VCEのサンプル版バイナリパッケージ(tar.gz,zip)を無償で提供しています。 無償ダウンロードをご利用いただくためには、 まずライセンス文書を読んでいただき、同意をされてからユーザー登録をして頂き、 その後自動的にダウンロード用のURLをメールで返信いたします。 この方法はVCEをご利用いただく上でコミュニティエンジンが的確な サポートをしていくために必要ですので、ご協力をお願いします。


  1. ライセンス確認/ユーザー登録
  2. 返信メールにてダウンロードURLを受信
  3. そのURLにてダウンロード

メーリングリスト

コミュニティエンジンでは、VCEの利用者コミュニティのための メーリングリストを運営しています。 メーリングリストに参加するには、 VCEのダウンロード用ユーザー登録画面 において、「メーリングリスト参加を希望」をチェックしてください。 現在の流量は月に数通程度で、 コミュニティエンジンが発信しているVCEに関するニュースが 中心となっています。



ロードマップ

 VCE は休むことなく進歩を続けています。
現在、VCEに追加中/実験中の機能は以下の通りです。
  • 帯域制限
    VCE全体、コネクション単位、仮想回路単位の、 各粒度における使用帯域制限をサポートします。
  • Dos/DDos 攻撃対策
    IPアドレスや頻度、通信内容にもとづく DoS 対策コードを内蔵します。
  • リモート・モニタリング機構
    Win32 環境で動作する、 複数のVCEサーバアプリケーションの監視ツールを提供します。
  • ストリーミング、IPv6への対応
  • 共有メモリルーチンの削除と公開
    共有メモリ関数群(mshare)はVCEの本来のレイヤーから逸脱気味であるため、 次期バージョンでは削除するとともに、公開する予定です。

VCE の開発では、エクストリームプログラミング手法でいうところの、 「短期間リリース」と「テスト最優先」の態度を保っています。
